📖 HTML Reference: The Skeleton Parts

In HTML, we don't just write text; we wrap text in Elements. An element is made of a Tag, Attributes (settings), and Content.

1. The Heading Tags (<h1> to <h6>)

Think of these like the "Font Size" presets in Microsoft Word or Google Docs.

How it works: <h1> is the most important (and largest). <h6> is the least important (and smallest).

Anatomy: <h1>Past Tense Pressure</h1>

Components: * <h1>: The opening tag (Start being a big title).

Past Tense Pressure: The content (The text you see).

</h1>: The closing tag (Stop being a title).

2. The Button Tag (<button>)

Buttons are built-in "clickable" areas.

How it works: It creates a box that reacts when clicked. By itself, it doesn't do anything—we use JavaScript later to tell it what to do.

Anatomy: <button>Save Key</button>

Components: The text between the tags is what is written on the physical button.

3. The Input Tag (<input>)

This is the most powerful tag in your assignment. It is unique because it is Self-Closing (it doesn't need a </input> tag).

Attributes (The "Parameters"): This is where you change how the box behaves.

type="text": A standard box for names or answers.

type="password": A box that turns characters into dots •••• (perfect for secret API keys).

placeholder="Type here...": Light gray text that disappears when you start typing.

Anatomy: <input type="text" placeholder="Student Answer">
